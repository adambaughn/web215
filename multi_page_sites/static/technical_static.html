Navigation Bar

The navigation bar used to navigate this page is simple to create.

Here are some of the main elements of this navigation bar.

  • The navigation links are contained inside a nav tag on the html page. The nav element defines a set of navigation links and it is intended for a major block of navigation links.
  • The position of the navigation bar is fixed so it is always visible to the viewer as they scroll through the page. The following code should be added to the id for your nav tag inside the CSS:
  • nav-bar { position: fixed; }
  • The entire block containing each link is turned into a link. To set the entire block as a link the following code should be added to the the class for you anchor tags inside the CSS:
  • .nav-link { display: inline-block; height: 100%; width: 100%; }
  • The the underline was removed from the links in the navigation bar by setting the text decoration to none. Use the following code on the anchor tag class for the navigation link inside your CSS:
  • .nav-link { text-decoration: none; }
  • Borders were added to div id tags for each link in the navigation bar.
  • This navigation bar is using margin-top at -9% for each div id in the navigation bar.
  • The size of each block containing the navigation links was set on the class for the links in the CSS. This navigation bar uses a width of 300px and a height of 40px for each block.

Creating the navigation bar was simple. Positioning everything is the most time consuming part.

Main Content

The main content of the page would be a mess without the power of CSS.

Here is some of the CSS used to display the main content of the page.

  • The section class for the main content of the page uses a top, top, right, bottom, and left margin with 3%.
  • The id for the main tag uses a margin-top with 2% and margin-left with 15.5%.
  • A code tag was added to display code on the page. The code tag in the CSS uses a 10px padded border. Setting margins on your code tag won't work in a list so you'll also need to add br tags to your HTML if you use a border with padding. This will prevent your borders from running over other items in your list.
  • The background color is #F0FFFF.
  • The font color is #000048.
Dynamic Index Page

This page is being displayed through my dynamic index page.

Here are more details about my dynamic index page.

  • My dynamic page uses links on the left to enable users to change the content of the page.
  • Changing content on my dynamic page uses less data and loads faster than viewing the same content on multiple static pages because I am only changing the content that needs to change.
  • Dynamic pages make it easier to check the consistency in the layout for the content of the pages.
  • Updating my dynamic pages are easier than updates to my static pages.
  • Creating the dynamic page was a little more difficult than creating static pages from a template but the small amount of additional time invested in creating dynamic pages is outweighed by their advantages.

Dynamic Page HTML

Javascript alone can't do all the work. It needs help from HTML to get the job done.

Here's a look at some of the HTML working behind the scenes to create this page dynamic.

  • The first step is to make sure you define your script source unless you're putting your Javascript on the same page as the HTML. Having HTML, CSS, or Javascript can get messy as more content is added so I prefer to keep them separate.
  • I placed the links for my dynamic index page inside a div without a nav tag. Each link uses a button with onclick equal to the function that calls the page. Here is the HTML that calls the function I showed in a previous example.
  • button type="button" id="introduction" class="buttons" onclick="loadIntroduction()">Introduction
  • After a user activates the function with onclick, content is loaded into my div. I gave my div an id called "dynamic_content" with some placeholder text between the div tags.
  • The page that is loaded into the div called "dynamic_content" is much simpler than a normal html page. I only need to create anything that I would put inside a main tag. No head, header, or footer is needed.
